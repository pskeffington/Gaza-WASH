# Verification Audit

## Current verification status

The current repository state contains a first-pass public-data extraction and scoring workflow. The extraction is based on public OCHA situation-report indicators and is suitable for method development, preliminary tabulation, and manuscript drafting.

## Verified public-source layer

The following source layer has been used in the populated extraction table:

- OCHA humanitarian situation report dated 2026-04-17.
- OCHA humanitarian situation report dated 2026-05-15.

The extracted indicators include displacement-site environmental hazards, WASH service stress, generator dependence affecting WASH facilities, operational constraints, and selected public-health service-capacity signals.

## Pending verification layer

The following source layers remain pending and should be added before final manuscript submission:

- WHO or Health Cluster facility-functionality data.
- HDX health-facility coordinate data.
- HDX or OCHA boundary data.
- WASH Cluster dashboard values where public export or stable date-stamped capture is available.

## Current analytic limitations

- Scores are aggregate or semi-aggregate, not household-level.
- Displacement-site indicators are reported as site-level conditions and should not be converted into individual risk estimates.
- Khan Younis has area-specific sewage flooding evidence, but some domain scores still rely on Gaza-wide contextual constraints.
- The current index is for prioritization and hypothesis generation, not causal inference.
- Dashboard-derived values should not be used unless exact date, access method, and variable definition are preserved.

## Manuscript language rule

Use: public-health risk signal, convergence of risk domains, prioritization, stratification, decision support.

Avoid: caused, proven, attributable mortality, disease incidence, intervention effect, definitive burden estimate.
